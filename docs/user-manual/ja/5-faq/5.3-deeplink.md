# 5.3 ディープリンクプロトコル

Agent Switch は `agentswitch://` リンクによるプロバイダー設定のワンクリックインポートに対応しています。

## 対応リソース

現在対応しているのはプロバイダーのインポートのみです：

```text
agentswitch://v1/import?resource=provider&app={app}&name={name}&...
```

`resource=mcp`、`resource=prompt`、`resource=skill` のリンクは拒否されます。

## 主なパラメータ

| パラメータ | 必須 | 説明 |
|-----------|------|------|
| `resource` | はい | `provider` 固定 |
| `app` | はい | `claude`、`claude-desktop`、`codex`、`gemini`、`opencode`、`openclaw`、`hermes` |
| `name` | はい | プロバイダー名 |
| `endpoint` | いいえ | API エンドポイント URL |
| `apiKey` | いいえ | API Key |
| `homepage` | いいえ | プロバイダーの Web サイト |
| `model` | いいえ | 既定モデル |
| `notes` | いいえ | メモ |
| `icon` | いいえ | アイコン |
| `config` | いいえ | Base64 エンコードされたプロバイダー設定 |
| `configFormat` | いいえ | `json` または `toml` |
| `configUrl` | いいえ | リモートプロバイダー設定 URL |
| `enabled` | いいえ | インポート後に有効化するか |

## 例

```text
agentswitch://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

リンクを開くと、Agent Switch は確認ダイアログを表示してからプロバイダーをインポートします。
