# 5.1 設定ファイルの説明

CC Switch は既定でデータをローカルに保存します。主なデータは、プロバイダー設定、Gateway 状態、使用量ログ、デバイス設定です。

## CC Switch データディレクトリ

既定の場所：

```text
~/.cc-switch/
├── cc-switch.db      # SQLite データベース
├── settings.json     # デバイス固有の設定
├── config.json       # 旧プロバイダー設定との互換ファイル
└── backups/          # ローカルバックアップ
```

`cc-switch.db` には、プロバイダー、エンドポイント、フェイルオーバー状態、使用量ログ、価格データ、プロキシ設定、同期メタデータが保存されます。

## クライアント設定ディレクトリ

プロバイダーを切り替える、または Gateway ルーティングを有効にすると、CC Switch は対象クライアントへ設定を書き込みます。

| クライアント | 既定ディレクトリ |
|-------------|------------------|
| Claude Code | `~/.claude/` |
| Claude Desktop | OS ごとの Claude Desktop 設定ディレクトリ |
| Codex | `~/.codex/` |
| Gemini CLI | `~/.gemini/` |
| OpenCode | `~/.config/opencode/` |
| OpenClaw | `~/.openclaw/` |
| Hermes | `~/.hermes/` |

これらのディレクトリは設定画面で上書きできます。

## インポート、エクスポート、同期

データエクスポートは SQL バックアップを作成し、以下を含みます：

- プロバイダー設定
- Gateway とプロキシ設定
- フェイルオーバーとストリームチェック状態
- 使用量ログと価格データ
- アプリ設定

WebDAV と S3 同期は同じデータベーススナップショット形式を使用し、互換性確認用の manifest を含みます。
