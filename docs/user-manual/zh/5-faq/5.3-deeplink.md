# 5.3 深度链接协议

Agent Switch 支持通过 `agentswitch://` 链接一键导入供应商配置。

## 支持的资源类型

当前仅支持导入供应商：

```text
agentswitch://v1/import?resource=provider&app={app}&name={name}&...
```

`resource=mcp`、`resource=prompt`、`resource=skill` 会被拒绝。

## 常用参数

| 参数 | 必填 | 说明 |
|------|------|------|
| `resource` | 是 | 必须为 `provider` |
| `app` | 是 | `claude`、`claude-desktop`、`codex`、`gemini`、`opencode`、`openclaw` 或 `hermes` |
| `name` | 是 | 供应商名称 |
| `endpoint` | 否 | API 端点 URL |
| `apiKey` | 否 | API Key |
| `homepage` | 否 | 供应商官网 |
| `model` | 否 | 默认模型 |
| `notes` | 否 | 备注 |
| `icon` | 否 | 图标 |
| `config` | 否 | Base64 编码的供应商配置 |
| `configFormat` | 否 | `json` 或 `toml` |
| `configUrl` | 否 | 远程供应商配置 URL |
| `enabled` | 否 | 导入后是否启用 |

## 示例

```text
agentswitch://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

打开链接后，Agent Switch 会先显示确认弹窗，再导入供应商。
