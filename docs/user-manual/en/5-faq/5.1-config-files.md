# 5.1 Configuration Files

Agent Switch keeps user data local by default. The main data categories are provider configuration, Gateway state, usage logs, and device settings.

## Agent Switch Data Directory

Default location:

```text
~/.agent-switch/
├── agent-switch.db      # SQLite database
├── settings.json     # Device-local settings
├── config.json       # Legacy provider config compatibility file
└── backups/          # Local backup files
```

`agent-switch.db` stores providers, endpoints, failover state, usage logs, pricing data, proxy settings, and sync metadata.

## Client Config Directories

Agent Switch writes provider settings to the target client when you switch providers or enable Gateway routing.

| Client | Default directory |
|--------|-------------------|
| Claude Code | `~/.claude/` |
| Claude Desktop | platform-specific Claude Desktop config directory |
| Codex | `~/.codex/` |
| Gemini CLI | `~/.gemini/` |
| OpenCode | `~/.config/opencode/` |
| OpenClaw | `~/.openclaw/` |
| Hermes | `~/.hermes/` |

Directory overrides can be configured in Settings.

## Import, Export, and Sync

Data export creates a SQL backup containing:

- Provider configurations
- Gateway and proxy settings
- Failover and stream check state
- Usage logs and pricing data
- App settings

WebDAV and S3 sync use the same database snapshot format, plus a manifest for compatibility checks.
