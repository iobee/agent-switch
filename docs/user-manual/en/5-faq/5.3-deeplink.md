# 5.3 Deep Link Protocol

CC Switch supports `ccswitch://` links for one-click provider import.

## Supported Resource

Only provider import is supported:

```text
ccswitch://v1/import?resource=provider&app={app}&name={name}&...
```

Links using `resource=mcp`, `resource=prompt`, or `resource=skill` are rejected.

## Common Parameters

| Parameter | Required | Description |
|-----------|----------|-------------|
| `resource` | Yes | Must be `provider` |
| `app` | Yes | `claude`, `claude-desktop`, `codex`, `gemini`, `opencode`, `openclaw`, or `hermes` |
| `name` | Yes | Provider name |
| `endpoint` | No | API endpoint URL |
| `apiKey` | No | API key |
| `homepage` | No | Provider website |
| `model` | No | Default model |
| `notes` | No | Notes |
| `icon` | No | Icon |
| `config` | No | Base64-encoded provider configuration |
| `configFormat` | No | `json` or `toml` |
| `configUrl` | No | Remote provider configuration URL |
| `enabled` | No | Whether to enable after import |

## Example

```text
ccswitch://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

After opening the link, CC Switch shows a confirmation dialog before importing the provider.
